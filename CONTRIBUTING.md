# Contributing Guidelines

Thank you for your interest in contributing to this project! To ensure an efficient and productive collaboration, please follow these guidelines:

1. **Open an Issue First**  
   Before making changes, please open an issue to discuss what you would like to change or add. This helps avoid duplicate work and gives everyone a chance to provide feedback.

2. **Fork the Repository and Create a Branch**  
   Fork the repository, then create a new branch with a descriptive name (e.g., `feature/unit-converter` or `bugfix/input-validation`).

3. **Write Clear, Concise Commit Messages**  
   Make your commit messages informative and meaningful. Briefly describe what your changes do and reference relevant issues if applicable.

4. **Follow the Existing Coding Style**  
   Keep your code consistent with the project's formatting, structure, and naming conventions. If in doubt, check existing code for reference.

5. **Test Before Submitting a Pull Request**  
   Make sure your changes work as expected and do not break existing functionality. Add or update tests if relevant.

6. **Add Anything you want and feel free to get involved but don't ruin it for me that's all I ask*
   
# Thank you for helping improve this project!
